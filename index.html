Piwi DashboardSelf-hosted Playwright observability

Understand and fix failures — don't just watch them. Failure clustering, flaky-test scoring, locator healing, and optional AI diagnosis on top of your Playwright results. Self-hosted, no SaaS, no lock-in.

Getting started
Live demo
GitHub
🧩

Failure clustering

Identical failures are grouped across specs by an error fingerprint, so one root cause is one thing to triage — not fifty scattered red tests.

🤖

AI failure diagnosis

Optional, opt-in analysis grounded in your actual git diff since the last green run — with suggested-fix patches validated server-side. Runs against your own provider (or a local model); nothing leaves your server unless you configure it.

📉

Flaky-test analytics

Composite flakiness score with root-cause classes (timing, network, assertion…) and CI-cost impact ranking, so you fix the flakes that actually waste the most CI minutes first.

🩹

Locator healing

When a locator breaks, get ranked replacement locators captured from prior passing runs — with a convention-preserving recommended fix and a data-testid nudge when nothing is stable.

🎬

Self-hosted trace viewer

Open the full Playwright trace viewer straight from a failure — bundled and served by the dashboard, so traces never leave your server.

🔔

Notifications & alerts

Email, Slack, webhook, and in-browser notifications for failed runs and new failure clusters, with per-project subscriptions, filters, and digest mode.

⚡

Live run streaming

Watch a run update in real time over SSE as each test finishes — no polling, no waiting on CI to upload an artifact.

📈

Performance tracking

Step-level timing, avg/P90 duration trends, slowest-tests analysis, and side-by-side run comparison.

🌐

Network & Web Vitals

Slow API endpoints grouped by method and normalized route, plus Core Web Vitals (TTFB, FCP, CLS…) with color-coded thresholds.

🔌

Drop-in reporter

Add one reporter to your Playwright config and results, HTML reports, and traces upload automatically after each run.

🤝

MCP server for AI agents

Query runs, failures, clusters, and flaky tests from Claude Code or any MCP client — bring test health into your coding agent.

🐳

Self-hosted, your data

One ~400 MB Docker image, SQLite or PostgreSQL, local or S3-compatible storage, optional role-based auth. Zero telemetry — nothing phones home.
